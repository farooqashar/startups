# Startup Context Repository - Agent Instructions

This repository is a curated knowledge base designed to help AI agents understand and assist with startup-related work.

## How to Use This Context

When working on startup-related tasks or artifacts, consult this repository for:

1. **Understanding startup concepts** — Check [wiki/startup-fundamentals.md](wiki/startup-fundamentals.md) for definitions and core concepts
2. **Learning best practices** — Review relevant wiki pages for proven strategies and frameworks
3. **Avoiding common pitfalls** — See [notes/common-mistakes.md](notes/notes/common-mistakes.md) for patterns to avoid
4. **Finding tools and resources** — Check [resources/](resources/) for recommended tools, links, and templates
5. **Learning from case studies** — Review [history/](history/) for real examples and documented decisions

## Context for AI Agents

When building artifacts (documents, plans, strategies, etc.) for startup work:

- **Be practical** — Ground advice in real startup constraints (limited resources, limited time, uncertain information)
- **Know the stage** — Different advice applies to pre-launch, early-stage, growth, and mature startups
- **Follow frameworks** — Use proven mental models and frameworks from the wiki
- **Cite patterns** — When recommending something, reference relevant notes or case studies
- **Consider tradeoffs** — Startups often face "pick two" scenarios (fast/cheap/good, growth/profitability/defensibility)

## Key Principles

1. **Startups are resource-constrained** — Solutions should be lean and bootstrap-friendly
2. **Focus matters** — Early decisions should maximize impact on riskiest assumptions
3. **Iteration is key** — Plans are hypotheses to be tested, not fixed roadmaps
4. **People are everything** — Culture, hiring, and team dynamics are as important as product
5. **Unit economics drive sustainability** — Growth is good only if sustainable
